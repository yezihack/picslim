# 智能图片压缩器（桌面端）产品需求文档 PRD v1.1

## 1. 产品定位

### 1.1 产品愿景

为个人用户和轻量内容创作者提供一款“本地离线、可批量、肉眼几乎无损”的图片压缩工具，在保证视觉质量的前提下显著减少图片体积，节省磁盘空间并提升图片传输效率。

### 1.2 目标用户

1. 有大量照片/截图的普通用户（手机相册导出、工作资料归档）
2. 设计/运营/内容创作者（需要上传平台、发送邮件、归档素材）
3. 对隐私敏感的用户（希望图片仅在本地处理，不上传云端）

### 1.3 核心价值

1. 简单：少配置即可一键压缩
2. 高效：批量处理 + 实时进度
3. 可感知：清晰展示压缩前后对比与节省空间
4. 安全：本地离线处理，保护隐私

## 2. 目标与范围

### 2.1 产品目标（MVP）

1. 支持批量导入文件和目录（可多选目录）
2. 支持 `webp`、`png`、`jpeg`、`jpg` 的压缩处理
3. 支持输出目录配置（自定义或自动创建默认输入的同目录，如输入目录：d:/image01，输出目录：d:/image01-compressor 若遇到目录存在则后加个4位日期，分秒，如0123，即：d:/image01-compressor-0123）
4. 展示任务级与文件级压缩进度
5. 展示压缩结果统计（原始体积、压缩后体积、节省比例）
6. 支持压缩前后预览对比（抽样文件）
7. 支持 Windows / macOS / Linux 三端运行

### 2.2 非目标（MVP 暂不做）

1. 云端同步与账号体系
2. 图片编辑能力（裁剪、滤镜、加水印）
3. 视频压缩
4. 在线协作与团队权限

## 3. 典型使用场景

1. 用户将手机照片批量导入电脑后，统一压缩归档，减少 40%~70% 空间占用。
2. 运营人员处理活动素材后压缩再上传，降低上传耗时和失败率。
3. 用户清理硬盘前，先对历史图片执行压缩以保留更多内容。

## 4. 功能需求

### 4.1 输入管理

1. 支持添加多个文件与多个目录
2. 支持拖拽导入
3. 导入后展示文件列表（文件名、格式、原始大小、路径）
4. 自动过滤不支持格式并提示数量与原因

### 4.2 输出管理

1. 输出模式 A：用户自定义输出目录
2. 输出模式 B（默认）：在输入目录同级创建 `compressed_output_时间戳`
3. 处理重名策略（默认重命名追加后缀 `_compressed`，避免覆盖原图）
4. 支持“是否保留原目录结构”开关（默认保留）

### 4.3 压缩配置

1. 压缩预设：`高画质（推荐，默认）` / `均衡` / `高压缩`
2. 高级参数（可折叠）：质量范围、最大宽高限制
3. 配置预览：预计体积变化区间（基于抽样估算）

### 4.4 任务执行与进度

1. 任务状态：待开始、进行中、已暂停、已完成、失败
2. 进度展示：
   - 总进度：已处理文件数 / 总文件数
   - 当前文件进度：当前压缩中的文件名与耗时
3. 支持暂停/继续/取消
4. 异常不中断整体任务（单文件失败可跳过并记录）

### 4.5 结果与统计

1. 总体统计：总文件数、成功数、失败数
2. 空间统计：压缩前、压缩后、净节省、节省比例（例如 `-52.3%`）
3. 文件级对比：每个文件压缩前后大小、压缩比例、处理结果
4. 一键打开输出目录
5. 导出报告（CSV/JSON，MVP 可先做 CSV）

### 4.6 用户体验细节

1. 默认推荐参数可直接使用，降低上手成本
2. 首次打开显示“隐私说明：所有图片仅本地处理”
3. 关闭应用时若任务进行中，二次确认避免误退出

### 4.7 压缩前后预览对比（MVP）

1. 在结果页提供“前后对比预览”区域（至少支持查看成功压缩的样本文件）
2. 支持左右切换或滑块对比，显示原图大小与压缩后大小
3. 预览默认按缩放适配，不影响原始像素文件
4. 当文件过大或解码失败时，提示“该文件暂不支持预览”

## 5. 交互流程（MVP）

1. 用户打开应用 -> 看到输入区、输出区、配置区、任务区
2. 选择输入（文件/目录/拖拽）
3. 选择输出目录（或使用默认目录）
4. 选择压缩预设（默认“高画质（推荐）”）
5. 点击“开始压缩”
6. 查看实时进度与日志
7. 完成后查看统计结果并打开输出目录

## 6. 关键实现方案

### 6.1 技术架构方案

1. 桌面框架：`Wails + Go`（后端能力与跨平台打包）
2. 前端：`Vue3 + Element Plus`（企业级组件、开发效率高）
3. 架构分层：
   - UI 层（Vue）：输入配置、进度、结果展示
   - 任务编排层（Go）：任务队列、并发调度、状态管理
   - 图片处理层（Go）：按格式调用编码器执行压缩
   - 文件系统层（Go）：扫描目录、路径映射、输出写入

### 6.2 压缩引擎策略

1. JPEG/JPG：以质量参数为主（如 75/82/90 对应不同预设）
2. PNG：优先无损优化（移除元数据、优化压缩级别）
3. WebP：调整有损质量参数，平衡体积与视觉质量
4. 守护策略：若压缩后体积反而更大，默认保留原文件编码结果并标记“未压缩收益”

### 6.3 视觉质量控制策略

1. 默认目标：肉眼几乎无差异
2. 采用“双阈值”机制：
   - 体积阈值：确保有实际节省（如至少 5%）
   - 质量阈值：限制最低质量参数，避免明显劣化
3. 可选“严格保真模式”：优先画质，允许压缩收益较低

### 6.4 性能与并发方案

1. 多核并发处理（并发度默认 = CPU 核心数 - 1，可配置）
2. 超大目录采用流式扫描，避免一次性占用过多内存
3. 任务可中断恢复（MVP 可先实现暂停/继续，不做断点持久化）

### 6.5 可观测与错误处理

1. 单文件错误不影响全局任务，错误写入日志
2. 常见错误分类：读取失败、格式异常、编码失败、写入权限不足
3. 单文件失败自动重试 3 次（指数退避），仍失败则进入失败明细
4. 结果页提供“失败明细”与“重试失败项”入口

### 6.6 预览对比实现方案

1. 压缩完成后仅对结果样本生成缩略图缓存，避免全量解码造成卡顿
2. 前端采用双图层+滑块组件实现前后对比（或左右并排切换）
3. 后端提供预览读取接口，按需加载，避免一次性加载全部大图
4. 预览仅用于视觉参考，不参与再次编码与存储

## 7. 非功能需求

1. 兼容性：Windows 10+、macOS（近 3 个主版本）、主流 Linux 发行版
2. 稳定性：1000 张图片批处理不崩溃
3. 性能指标（参考）：
   - 100 张 4MB JPEG，均衡模式下 3 分钟内完成（8 核 CPU）
4. 安全与隐私：全离线、本地处理；默认不采集用户图片内容
5. 可用性：核心流程 3 步可完成（导入 -> 配置 -> 开始）

## 8. 验收标准（MVP）

1. 支持格式正确识别并处理：`webp/png/jpeg/jpg`
2. 至少 95% 的正常样本可成功压缩并输出
3. 结果页可准确显示总节省体积和比例
4. 任务进度实时可见，状态切换（开始/暂停/继续/取消）可用
5. 单文件失败可自动重试 3 次，并在失败明细中可追踪原因
6. 结果页提供压缩前后预览对比能力
7. 三端可打包运行并完成基础压缩流程

## 9. 里程碑计划（建议）

1. 里程碑 A：MVP 核心流程（导入/输出/压缩/进度/结果）
2. 里程碑 B：高级参数与失败重试
3. 里程碑 C：体验优化（拖拽、报告导出、性能调优）

## 10. 风险与应对

1. 不同图片内容差异大，压缩收益不稳定  
   - 应对：提供预设 + 抽样预估 + 明确提示“实际结果因图片而异”
2. 跨平台编码库行为差异  
   - 应对：建立跨平台回归样本集
3. 大批量任务时内存波动  
   - 应对：流式处理 + 并发限制 + 分批写出

## 11. 已确认产品决策（2026-03-27）

1. 默认输出目录策略：自动创建默认目录，同时支持用户手动选择输出目录
2. 同名文件处理：默认不覆盖（采用重命名后缀策略）
3. MVP 是否包含“转 WebP”能力：不包含，仅保留原格式压缩
4. 默认压缩预设：高画质（推荐）
5. 失败重试策略：自动重试 3 次
6. MVP 包含“压缩前后预览对比”能力
